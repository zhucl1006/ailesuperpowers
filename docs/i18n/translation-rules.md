# 翻譯規則（全量繁體）

## 必須翻譯

- Markdown 自然語言段落
- 標題、列表、說明文字、註解（不影響語法者）
- 使用者可見提示訊息

## 禁止翻譯

- 前題的`name:` 值
- 程序碼識別符、函式名、變量名、JSON key
- 指令、參數、路徑、URL、位址空間
- 程式碼區塊內容（```...```）

## 結構保護規則

1. 不修改Markdown連結語法：`[文字](url)`。
2. 不刪除或改寫反引號包住的內容：`` `...` ``。
3. 不更改 YAML frontmatter 邊界：`---`。
4. 不修改 shell/js 的數據結構鍵名與輸出欄位名。

## 品質檢查

- frontmatter檢查：`name:` 保持原值
- 結構檢查：連結、程式碼塊、表格分隔符保持完整
- 行為檢查：`hooks/session-start.sh`JSON可被`jq` 解析
