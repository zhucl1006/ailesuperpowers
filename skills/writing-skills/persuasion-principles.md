# 技能設計的說服原則

＃＃ 概述

法學碩士遵循與人類相同的說服原則。瞭解這種心理可以幫助您設計更有效的技能 - 不是操縱，而是確保即使在壓力下也能遵循關鍵實踐。

**研究基礎：** Meincke et al. (2025) 透過 N=28,000 次人工智慧對話測試了 7 項說服原則。說服技巧使遵守率提高了一倍以上 (33% → 72%, p < .001)。

## 七項原則

### 1. 權威
**它是什麼：** 對專業知識、證書或官方來源的尊重。

**技能如何發揮作用：**
- 命令式語言：“你必須”、“從不”、“總是”
- 不可協商的框架：“沒有例外”
- 消除決策疲勞與合理化

**何時使用：**
- 紀律執行技能（TDD、驗證要求）
- 安全關鍵實踐
- 建立最佳實踐

**例子：**```markdown
✅ Write code before test? Delete it. Start over. No exceptions.
❌ Consider writing tests first when feasible.
```

### 2. 承諾
**它是什麼：** 與之前的行動、聲明或公開聲明保持一致。

**技能如何發揮作用：**
- 需要公告：“公告技能使用情況”
- 強制明確的選擇：“選擇 A、B 或 C”
- 使用追蹤：TodoWrite 作為清單

**何時使用：**
- 確保技能實際遵循
- 多步驟流程
- 問責機制

**例子：**```markdown
✅ When you find a skill, you MUST announce: "I'm using [Skill Name]"
❌ Consider letting your partner know which skill you're using.
```

### 3.稀缺性
**它是什麼：** 由於時間限製或可用性有限而產生的緊迫性。

**技能如何發揮作用：**
- 有時間限制的要求：“在繼續之前”
- 順序依賴：“緊接在 X 之後”
- 防止拖延

**何時使用：**
- 立即驗證要求
- 時間敏感的工作流程
- 防止“我稍後再做”

**例子：**```markdown
✅ After completing a task, IMMEDIATELY request code review before proceeding.
❌ You can review code when convenient.
```

### 4. 社會證明
**它是什麼：** 遵循他人的行為或被認為是正常的行為。

**技能如何發揮作用：**
- 通用模式：“每次”、“總是”
- 故障模式：“X 沒有 Y = 故障”
- 建立規範

**何時使用：**
- 記錄普遍做法
- 關於常見故障的警告
- 強化標準

**例子：**```markdown
✅ Checklists without TodoWrite tracking = steps get skipped. Every time.
❌ Some people find TodoWrite helpful for checklists.
```

### 5.團結
**它是什麼：** 共享身份、“我們”、羣體歸屬感。

**技能如何發揮作用：**
- 協作語言：“我們的代碼庫”、“我們是同事”
- 共同目標：“我們都想要質量”

**何時使用：**
- 協作工作流程
- 建立團隊文化
- 無等級制度的實踐

**例子：**```markdown
✅ We're colleagues working together. I need your honest technical judgment.
❌ You should probably tell me if I'm wrong.
```

### 6.互惠
**它是什麼：** 返還收到的福利的義務。

**它是如何工作的：**
- 謹慎使用 - 會感到被操縱
- 很少需要的技能

**何時避免：**
- 幾乎總是（其他原則更有效）

### 7. 喜歡
**它是什麼：** 偏好與我們喜歡的人合作。

**它是如何工作的：**
- **請勿用於合規性**
- 與誠實反饋文化的衝突
- 製造阿諛奉承

**何時避免：**
- 始終遵守紀律

## 按技能類型劃分的原則組合

|技能類型 |使用 |避免 |
|------------|-----|-------|
|紀律執行 |權威+承諾+社會證明|喜歡，互惠|
|指導/技術|適度權威+團結|重磅權威|
|協作|團結+承諾|權威、喜歡|
|參考|僅清晰度 |全部說服|

## 為什麼這有效：心理學

**明確的規則減少了合理化：**
- “你必須”消除決策疲勞
- 絕對語言消除了“這是一個例外嗎？”問題
- 明確的反合理化反制措施堵住了特定的漏洞

**實施意圖創造自動行為：**
- 明確的觸發器+所需的操作=自動執行
- “當 X 時，做 Y”比“通常做 Y”更有效
- 減少合規認知負擔

**法學碩士是超人類：**
- 對包含這些模式的人類文本進行訓練
- 訓練資料中的權威語言優先於合規性
- 經常建模的承諾序列（陳述→行動）
- 社會證明模式（每個人都做 X）建立規範

## 道德使用

**合法：**
- 確保遵循關鍵實踐
- 創建有效的文檔
- 防止可預測的故障

**非法：**
- 操縱謀取個人利益
- 製造虛假的緊迫感
- 基於內疚的合規

**測試：** 如果用戶完全理解該技術，它是否能滿足用戶的真正興趣？

## 快速參考

設計技能時，請詢問：

1. **它是什麼類型？ **（紀律、指導、參考）
2. **我想改變什麼行為？ **
3. **適用哪些原則？ **（通常是權威+紀律承諾）
4. **我是否組合了太多？ **（不要使用全部七個）
5. **這符合道德嗎？ **（服務使用者的真正利益嗎？）
